# 青城两河漂流 - 售票后台管理系统

## 一、项目概况

**业务**：漂流景区售票后台，支持新建订单、打印门票、查询统计。

**技术栈**：

| 层 | 技术 |
|---|---|
| UI框架 | React 16.7 |
| 状态管理 | MobX 5（装饰器语法） |
| 组件库 | Ant Design 3.x |
| 路由 | react-router v3（hashHistory） |
| HTTP | axios，代理到后端服务 |
| 构建 | react-app-rewired + customize-cra |
| 样式 | 纯内联 style / className，无预处理器 |

## 五、新需求实现方案

### 需求1：手机尾号查询
- `storeOrder.js` 新增 `filterPhoneTail` observable + `setPhoneTailFilter` action
- `fiter` computed 增加尾号过滤：`item.phoneNumber.endsWith(filterPhoneTail)`
- `Table.jsx` 筛选区增加尾号输入框

### 需求2：押金是否已退还
- `isReback` 字段已存在，但需按"手机尾号分组"——同尾号多订单共享一次押金
- `storeOrder.js` 新增 computed `depositGroupMap`：以手机尾号为 key，记录该组是否有任意一条 `isReback=true`
- `Table.jsx` 增加"押金状态"列，展示该尾号组的退押金状态

### 需求3：导出月/日数据
- 安装 `xlsx`（SheetJS）库
- `storeOrder.js` 新增 `exportData()` 方法，按当前筛选结果生成 Excel
- `Table.jsx` 增加"导出"按钮

### 需求4：错票备注
- `InputBox` 新增 `errorNote` 字段（错票原因/跳号备注）
- `AddOrderModal.jsx` 增加备注输入框
- `Table.jsx` 增加"备注"列
- 后端接口需同步增加该字段

### 需求5：打印时间精确到分秒
- `storeOrder.js` `initInput` 中将 `time` 改为 `moment(Date.now()).format("YYYY-MM-DD HH:mm:ss")`
- `Printer.jsx` 解析时间时拆分到秒并展示

## 六、Sass 重构方案

1. 安装 `sass`（CRA 2.x 内置支持，只需装包）
2. 创建 `src/styles/` 目录：
   - `_variables.scss`（颜色、字号、间距变量）
   - `_layout.scss`（header、容器）
   - `_table.scss`（表格、筛选区）
   - `_modal.scss`（弹窗）
   - `_printer.scss`（打印表单，含 `@media print`）
3. 将组件内联 style 迁移到 `.scss`，className 统一 BEM 命名
